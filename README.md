# WORKMORE Assignment - 대출나라 리워크 프로젝트

## 배포 링크

- Live Demo: https://workmore-assignment.vercel.app/

## 실행 방법

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:3000`으로 접속하면 프로젝트를 확인할 수 있습니다.

---

## 프로젝트 개요

본 프로젝트는 기존 `대출나라` 서비스를 보다 간단명료하고 사용하기 쉬운 금융 정보 플랫폼으로 재해석한 프론트엔드 리워크 과제입니다.

기존 대출나라 사이트는 대출업체 정보, 광고, 금융 정보, 고객센터, 안전 관련 안내 등 다양한 정보를 제공하고 있었지만, 첫 화면에 많은 텍스트와 기능이 한꺼번에 노출되어 사용자가 어떤 기능을 먼저 사용해야 하는지 파악하기 어려운 구조였습니다.

이번 리워크에서는 완성도 높은 전체 서비스를 구현하는 것보다, 짧은 시간 안에 서비스의 핵심 문제를 파악하고 이를 UX/UI 관점에서 빠르게 개선하는 것에 집중했습니다. 특히 실제 사용자가 대출업체를 탐색하는 흐름을 중심으로 정보 구조를 재정리하고, 필요한 기능을 명확하게 분리하여 보다 직관적인 사용자 경험을 제공하는 것을 목표로 했습니다.

---

## 문제 정의

기존 대출나라 사이트에서 가장 크게 느껴진 문제는 정보의 양이 많음에도 불구하고, 사용자가 어떤 순서로 서비스를 이용해야 하는지 쉽게 이해하기 어렵다는 점이었습니다.

첫 페이지에는 대출업체 정보, 광고성 문구, 금융 정보, 고객센터 관련 내용이 동시에 노출되어 있었고, 이로 인해 사용자는 원하는 정보를 찾기 전에 많은 텍스트를 읽어야 했습니다.

또한 일반 금융서비스 사용자와 대출업체 사업자를 위한 정보가 한 화면 안에 섞여 있어, 사용자의 목적에 따른 명확한 동선이 부족하다고 판단했습니다.

### 주요 문제점

- 첫 페이지에 텍스트가 과도하게 많음
- 사용자가 서비스를 어떻게 이용해야 하는지 직관적으로 파악하기 어려움
- 주요 기능의 우선순위가 명확하지 않음
- 금융서비스 사용자와 대출업체 사업자의 정보가 혼재되어 있음
- 업체 탐색, 안전 정보, 금융 콘텐츠, 광고 등록 등의 기능이 명확하게 분리되어 있지 않음

## 서비스 분석

대출나라는 기본적으로 대출이 필요한 사용자에게 대출업체 정보를 제공하고, 동시에 대출업체에게는 광고 및 홍보 채널을 제공하는 플랫폼입니다.

따라서 이 서비스에는 크게 두 종류의 사용자가 존재한다고 판단했습니다.

### 일반 사용자

일반 사용자는 대출이 필요하거나 관련 정보를 찾고 있는 사람입니다. 이들은 사이트에 들어왔을 때 가장 먼저 신뢰할 수 있는 대출업체를 빠르게 찾고 싶어합니다.

일반 사용자에게 중요한 정보는 다음과 같습니다.

- 지역별 대출업체 정보
- 상품별 대출업체 정보
- 업체의 간단한 소개
- 업체 특징
- 전화번호 및 상담 연결
- 안전하게 대출업체를 이용하기 위한 안내
- 금융 관련 기본 정보

### 대출업체 사용자

대출업체 사용자는 플랫폼에 자신의 업체를 등록하거나 광고를 신청하려는 사업자입니다. 이들은 일반 사용자와는 다른 목적을 가지고 있기 때문에 별도의 동선으로 분리하는 것이 적절하다고 판단했습니다.

대출업체 사용자에게 필요한 기능은 다음과 같습니다.

- 광고 등록
- 업체 회원가입
- 광고 및 제휴 문의
- 업체 정보 등록 및 관리

## 해결 방향

이번 리워크에서는 모든 기능을 한 화면에 보여주기보다, 사용자의 목적에 따라 기능을 분리하고 우선순위를 재정리하는 방향으로 접근했습니다.

가장 중요한 사용자는 대출업체를 찾기 위해 방문한 일반 사용자라고 판단했습니다. 따라서 메인 화면에서는 사용자가 페이지에 들어오자마자 바로 업체를 찾을 수 있도록 검색과 탐색 기능을 중심에 배치했습니다.

또한 한 페이지에서 너무 많은 기능을 수행하지 않도록 구성하고, 각 페이지의 목적을 명확하게 분리했습니다.

### 핵심 개선 방향

- 일반 사용자의 업체 탐색 플로우를 최우선으로 설계
- 업체 목록과 업체 상세 페이지의 역할을 분리하여 정보 가독성 개선
- 업체 정보, 안전센터, 금융정보, 고객센터를 명확하게 분리
- 대출업체 사업자를 위한 광고 등록/문의 동선은 별도로 구성
- 한 페이지에서는 하나의 핵심 기능에 집중
- 과도한 설명 텍스트를 줄이고 카드, 칩, 섹션 단위로 정보 구조화
- 사용자가 지금 무엇을 해야 하는지 명확하게 인지할 수 있는 UI 구성

## 프로젝트에서 중요하게 본 점

이번 과제에서 가장 중요하게 본 것은 단순히 예쁜 화면을 만드는 것이 아니라, 기존 서비스의 문제를 파악하고 제한된 시간 안에서 현실적인 개선 방향을 제시하는 것이었습니다.

대출나라와 같은 정보 중심 플랫폼에서는 사용자가 원하는 업체를 빠르게 찾고, 상담 전 필요한 정보를 쉽게 이해하는 것이 중요하다고 판단했습니다. 그래서 메인 화면은 업체 탐색 중심으로 구성하고, 업체 상세 페이지는 선택한 업체 정보를 더 읽기 쉽게 확인할 수 있도록 정리했습니다.

기존 서비스에 없던 이자 계산기 기능도 추가했습니다. 사용자가 업체를 찾는 것에서 끝나는 것이 아니라, 대출 조건에 따른 예상 이자를 직접 확인해보며 더 신중하게 판단할 수 있도록 돕고자 했습니다.

금융 서비스 특성상 신뢰감도 중요하기 때문에, 과도하게 화려한 UI보다는 깔끔한 레이아웃과 명확한 정보 구조를 우선했습니다.

## 주요 기능 구성

### 업체 찾기

기존의 지역별 업체찾기, 상품별 업체찾기, 맞춤검색 기능을 통합적인 탐색 경험으로 재구성했습니다.

사용자는 지역, 상품, 상황과 같은 조건을 기반으로 원하는 대출업체를 빠르게 찾을 수 있습니다. 복잡한 텍스트 목록 대신 카드와 필터 중심의 UI를 사용하여 정보를 더 쉽게 비교할 수 있도록 설계했습니다.

### 업체 정보 카드

업체 카드에는 사용자가 업체를 판단하는 데 필요한 최소한의 정보만 우선적으로 노출했습니다.

표시 정보는 다음과 같습니다.

- 업체명
- 지역
- 한 줄 소개
- 업체 특징
- 전화번호 또는 상담 연결 버튼

텍스트가 길어질 경우 카드가 복잡해질 수 있기 때문에, 업체 특징이나 지역 정보는 칩 형태로 정리하고 핵심 CTA는 명확하게 분리했습니다.

### 업체 상세 페이지

업체 상세 페이지는 사용자가 특정 업체에 대해 더 자세한 정보를 확인하는 페이지로 구성했습니다.

기존에는 업체 정보가 텍스트 위주로 길게 나열되어 있어 핵심 정보를 빠르게 파악하기 어렵다고 판단했습니다. 이를 개선하기 위해 업체명, 지역, 주요 특징, 상담 정보, 상세 소개 등을 영역별로 나누고, 카드와 섹션 단위로 정리하여 가독성을 높였습니다.

목록 페이지에서는 간단한 비교가 가능하도록 하고, 상세 페이지에서는 업체에 대한 구체적인 정보를 안정적으로 확인할 수 있도록 역할을 분리했습니다.

### 안전센터

금융 서비스 특성상 사용자가 신뢰와 안전을 중요하게 고려한다고 판단했습니다.

따라서 안전센터는 사용자가 대출 서비스를 이용할 때 참고할 수 있는 안전 관련 정보를 모아둔 영역으로 구성했습니다.

예를 들어 불법 사금융 주의사항, 정식 등록 업체 확인 방법, 상담 전 확인하면 좋은 체크리스트 등을 제공할 수 있도록 설계했습니다.

### 금융정보

금융정보는 사용자가 대출을 선택하기 전에 기본적으로 이해해야 할 정보를 제공하는 콘텐츠 영역입니다.

기존의 금융뉴스, 금융팁, 안전 가이드 등을 커뮤니티성 정보로 묶되, 사용자가 부담 없이 읽을 수 있도록 카드 기반 콘텐츠 형태로 구성했습니다.

### 이자 계산기

기존 서비스에는 이자 계산기 기능이 없었지만, 사용자가 대출 상품이나 업체를 비교할 때 예상 이자를 직접 계산해볼 수 있으면 더 실질적인 도움이 될 것이라고 판단했습니다.

따라서 단순히 업체 정보를 보여주는 것에서 끝나지 않고, 사용자가 대출 금액과 기간을 기준으로 대략적인 이자 부담을 확인할 수 있는 계산기 기능을 추가했습니다. 이를 통해 사용자는 상담 전 스스로 조건을 가늠해볼 수 있고, 서비스는 단순 정보 제공을 넘어 사용자의 의사결정을 돕는 방향으로 확장될 수 있다고 생각했습니다.

### 고객센터

고객센터는 일반 사용자와 업체 사용자가 각각 필요한 도움을 빠르게 찾을 수 있도록 구성했습니다.

단순한 문의 페이지가 아니라, 자주 묻는 질문, 이용 안내, 문의하기 등의 기능을 명확히 분리하여 사용자가 불필요하게 헤매지 않도록 설계했습니다.

### 광고 등록 및 업체 문의

대출업체 사업자를 위한 동선은 일반 사용자 탐색 흐름과 분리했습니다.

일반 사용자는 업체를 찾는 데 집중하고, 업체 사용자는 광고 등록이나 입점 문의를 별도 페이지에서 진행할 수 있도록 구성했습니다.

## UX 설계 의도

이번 프로젝트의 핵심 UX 방향은 “사용자가 들어오자마자 무엇을 해야 하는지 알 수 있는 구조”를 만드는 것이었습니다.

기존 사이트처럼 많은 메뉴와 텍스트를 한 번에 보여주기보다, 사용자의 첫 번째 목적을 “업체 찾기”로 정의하고 이에 맞춰 메인 화면의 우선순위를 재배치했습니다.

### UX 설계 원칙

- 사용자는 사이트에 들어오자마자 업체를 찾을 수 있어야 함
- 검색과 필터는 복잡하지 않고 직관적이어야 함
- 일반 사용자와 업체 사용자의 동선을 분리해야 함
- 한 페이지에는 하나의 주요 목적만 담아야 함
- 금융 서비스이기 때문에 신뢰감과 안정감을 줄 수 있어야 함
- 정보는 길게 설명하기보다 카드, 칩, 버튼으로 구조화해야 함

---

## UI 설계 방향

UI는 기존 대출나라 서비스에서 사용되던 주황색 계열을 메인 컬러로 유지하되, 보다 정돈되고 현대적인 인상을 줄 수 있도록 재해석했습니다.

대출 서비스는 자칫 딱딱하거나 오래된 인상을 줄 수 있기 때문에, 기존 서비스의 브랜드 컬러는 살리면서도 과도한 색 사용은 줄이고, 따뜻하고 신뢰감 있는 화면을 만드는 방향으로 디자인했습니다.

### UI 디자인 원칙

- 대출나라의 기존 메인 컬러인 주황색 계열을 primary color로 활용
- 단일한 브랜드 컬러 중심으로 구성
- White, Off White, Warm Gray 배경을 활용하여 정보 가독성 확보
- 카드 기반 UI로 정보 덩어리를 명확하게 분리
- 칩 UI를 활용하여 지역, 상품, 상황 정보를 간결하게 표현
- CTA 버튼은 명확한 색상과 문구로 사용자의 행동을 유도
- 과도한 텍스트를 줄이고 시각적 위계를 통해 정보 전달

## 기술 선택 이유

### Next.js

Next.js는 React 기반의 프레임워크로, 페이지 라우팅과 프로젝트 구조를 빠르게 구성할 수 있다는 장점이 있습니다.

이번 과제는 짧은 시간 안에 서비스의 핵심 화면과 UX 흐름을 빠르게 구현하는 것이 중요했기 때문에, Next.js를 사용하여 초기 세팅 시간을 줄이고 화면 구현에 집중했습니다.

향후 실제 서비스로 확장될 경우, Next.js의 라우팅 구조를 기반으로 메인 페이지, 업체 찾기 페이지, 안전센터, 금융정보, 고객센터, 광고 등록 페이지 등을 명확하게 분리할 수 있습니다.

대출 정보 플랫폼은 사용자가 검색 엔진을 통해 유입될 가능성이 높은 서비스라고 판단했습니다. 따라서 Next.js의 SSR(Server Side Rendering) 능력을 활용하면 초기 HTML 응답 단계에서 주요 콘텐츠를 제공할 수 있고, 이는 검색 엔진이 페이지 내용을 이해하는 데 유리하게 작용할 수 있습니다. 업체 정보, 금융 정보, 안전 가이드와 같은 콘텐츠 중심 페이지는 SEO 관점에서도 중요한 자산이 될 수 있기 때문에, Next.js는 단순한 화면 구현뿐만 아니라 향후 서비스 확장성 측면에서도 적합하다고 판단했습니다.

### TypeScript

TypeScript는 컴포넌트 props, 데이터 모델, 필터 조건 등의 타입을 명확하게 정의할 수 있어 유지보수성과 안정성을 높일 수 있습니다.

업체 카드, 검색 조건, 지역/상품/상황 칩과 같은 UI 데이터는 구조가 반복적으로 사용되기 때문에, TypeScript를 사용하면 컴포넌트 간 데이터 전달 과정에서 발생할 수 있는 오류를 줄일 수 있습니다.

### Tailwind CSS

Tailwind CSS는 빠르게 UI를 구성하면서도 일관된 디자인 시스템을 유지하기에 적합합니다.

이번 프로젝트에서는 제한된 시간 안에 여러 화면과 컴포넌트를 구현해야 했기 때문에, Tailwind CSS를 활용하여 spacing, color, typography, radius 등의 스타일을 빠르게 적용했습니다.

또한 카드, 칩, 버튼, 섹션 레이아웃 등 반복되는 UI 패턴을 일관성 있게 구성할 수 있도록 했습니다.

### Pretendard

Pretendard는 한글 가독성이 좋은 웹 폰트이기 때문에, 금융 정보처럼 텍스트 전달이 중요한 서비스에 적합하다고 판단했습니다.

기존 사이트의 복잡한 텍스트 구조를 줄이더라도, 남아 있는 핵심 정보는 명확하게 읽혀야 하기 때문에 Pretendard를 적용했습니다.

### FSD 기반 구조 설계

이번 프로젝트는 FSD(Feature-Sliced Design)를 참고하여 폴더 구조를 설계했습니다.

모든 기능을 완성도 높게 구현하는 것도 중요하지만, 짧은 시간 안에 서비스의 핵심 화면을 빠르게 만들고 이후 확장 가능한 구조를 구성하는 것이 더욱 중요하다고 생각했습니다.

따라서 라우팅은 `app`, 페이지 단위 화면 조합은 `views`, 섹션 단위 UI는 `widgets`, 사용자 행동 중심 기능은 `features`, 서비스의 핵심 도메인 정보는 `entities`, 공통 UI와 유틸은 `shared`로 분리했습니다.

이를 통해 특정 페이지 안에 모든 코드를 작성하는 방식보다 컴포넌트의 책임을 명확히 나눌 수 있었고, 업체 카드, 검색 필터, 안전센터 섹션, 이자 계산기처럼 반복적으로 사용되거나 확장될 수 있는 요소를 더 재사용하기 쉽게 구성했습니다.

### 구조 설계 의도

- `app`: Next.js App Router 기반의 실제 라우팅 관리
- `views`: 각 페이지에서 사용되는 화면 단위 조합 관리
- `widgets`: 헤더, 푸터, 검색 영역, 안전센터 레이아웃처럼 여러 컴포넌트를 묶은 섹션 단위 UI 관리
- `features`: 이자 계산, 업체 검색, 피해 신고, 등록 업체 확인 등 사용자 행동 중심 기능 관리
- `entities`: 대출업체, 게시글, 공지, FAQ, 안전 정보처럼 서비스의 핵심 도메인 단위 관리
- `shared`: 버튼, 카드, 공통 유틸, 상수 등 여러 영역에서 재사용되는 공통 요소 관리

이 구조를 통해 페이지별 구현에만 의존하지 않고, 화면을 구성하는 요소를 역할별로 분리했습니다. 특히 업체 탐색, 업체 상세, 안전센터, 고객센터, 이자 계산기처럼 여러 화면에서 반복될 수 있는 UI와 기능을 재사용 가능한 단위로 나누는 데 집중했습니다.

## 아쉬운 부분 및 추가 발전사항

이번 프로젝트는 짧은 시간 안에 기존 서비스를 분석하고 핵심 UX/UI를 재구성하는 것에 집중했기 때문에, 실제 서비스 수준의 모든 기능을 완성하지는 못했습니다.

기업 캐릭터나 단순한 3D 모델링 이미지를 활용했다면 서비스의 분위기를 더 깔끔하고 세련되게 만들 수 있었을 것이라고 생각합니다. 예를 들어 안전센터, 고객센터, 업체 탐색 영역에 가벼운 3D 오브젝트나 브랜드 캐릭터를 배치했다면 텍스트 중심의 화면을 덜어내면서도 사용자에게 더 친근한 인상을 줄 수 있었을 것입니다. 하지만 제한된 시간 안에서는 정보 구조와 핵심 UI 구현에 우선순위를 두었기 때문에, 이러한 비주얼 요소까지 충분히 반영하지 못한 점이 아쉬웠습니다.

업체 리스트 영역에 무한 스크롤을 적용하지 못한 점도 아쉬운 부분입니다. 대출업체 정보는 사용자가 여러 업체를 연속적으로 탐색하고 비교하는 흐름이 중요하기 때문에, 페이지네이션보다 무한 스크롤이 모바일과 PC 모두에서 더 부드러운 탐색 경험을 제공할 수 있다고 판단했습니다. 하지만 다른 UI 구현을 우선해야 했기 때문에, 이번 작업 범위에서는 제외했습니다.
